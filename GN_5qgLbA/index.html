gridea 远程仓库设置

2023-09-11

序

简单总结一下我遇到的问题和解决方法。
方便我自己下次重新配置。

设置

平台

我自己选择 Github Pages ，各位可以自己选择其他的。
配置因该大同小异。

域名

默认勾选https，当然你也可以选择用http（我没用过http，不知效果。

这里是你 Github Pages 访问的地址。
默认是 Github名字.github.io 。（就是和仓库名称一致）
否则就是你自己设置的域名。
注：如果你使用了自己的域名，这里一定要设置成自己的域名（就是访问出来地址栏显示的域名）
否则会css之类的资源访问出错！

仓库名称

就是你的仓库名称，一般是你 Github名字.github.io 。

分支

Github 现在默认是 main 分支。

仓库用户名

就是你的用户名

邮箱

因该是可以随便设置的，我猜是用来 git 推上去时候需要的参数。

令牌

令牌创建地址：https://github.com/settings/tokens
注：

  • 你可以只勾选 repo 来提高安全性。
  • 你也可以勾选永久不过期。

CNAME

如果你设置了自己的域名，这里填你设置的域名，否则不填就行。

HTTP代理

不懂的话，默认就行。
